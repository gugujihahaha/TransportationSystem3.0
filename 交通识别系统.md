## 一、项目背景与网页定位

### 1.1 项目是什么？

这是一个**基于GPS轨迹数据的交通方式识别系统**。

简单来说：用户上传一段GPS轨迹数据（比如手机记录的出行轨迹），系统能自动识别这段轨迹是**步行、骑车、开车、坐公交、坐地铁还是坐火车**。

### 1.2 网页定位

| **目标用户** | 学术答辩评审、项目演示观众、潜在合作方 |

## 三、模型输入输出规范

### 3.1 Exp1 模型（最简单）

#### 输入格式

```python
# 输入：轨迹特征矩阵
# 形状：(seq_len, 9) 或 (batch_size, seq_len, 9)
# seq_len：轨迹点数量，建议50-100个点

trajectory_features = numpy.array([
    [lat, lon, speed, acceleration, bearing_change, distance, time_diff, total_distance, total_time],
    [lat, lon, speed, acceleration, bearing_change, distance, time_diff, total_distance, total_time],
    # ... 更多轨迹点
])
# 形状示例：(50, 9) 表示50个点，每个点9个特征
```

#### 9维轨迹特征详解

| 索引  | 特征名            | 说明   | 单位   | 计算方式          |
| --- | -------------- | ---- | ---- | ------------- |
| 0   | latitude       | 纬度   | 度    | 原始GPS数据       |
| 1   | longitude      | 经度   | 度    | 原始GPS数据       |
| 2   | speed          | 瞬时速度 | 米/秒  | 相邻两点距离÷时间差    |
| 3   | acceleration   | 加速度  | 米/秒² | 速度变化÷时间差      |
| 4   | bearing_change | 方向变化 | 度    | 相邻两段的航向角差值    |
| 5   | distance       | 点间距离 | 米    | Haversine公式计算 |
| 6   | time_diff      | 时间差  | 秒    | 相邻两点时间戳之差     |
| 7   | total_distance | 累计距离 | 米    | 从起点到当前点的总距离   |
| 8   | total_time     | 累计时间 | 秒    | 从起点到当前点的总时间   |

#### 输出格式

```python
# 调用预测
label, confidence = predictor.predict(trajectory_features)

# 返回值
label = "Walk"        # 字符串，预测的交通方式
confidence = 0.856    # 浮点数，置信度 (0-1)
```

#### 调用示例

```python
from predict import TrajectoryPredictor
import numpy as np

# 初始化（只需要模型文件）
predictor = TrajectoryPredictor("checkpoints/exp1_model.pth")

# 准备输入（50个轨迹点，每个点9维特征）
trajectory = np.array([...])  # shape: (50, 9)

# 预测
label, confidence = predictor.predict(trajectory)
print(f"预测结果: {label}, 置信度: {confidence:.2%}")
```

### 3.5 Exp5 模型（弱监督上下文增强）

#### 输入格式

```python
trajectory_features  # shape: (seq_len, 9)
```

#### 融合策略说明

Exp5采用弱监督约束策略，与Exp4的硬拼接不同：
- **Exp4（硬拼接）**: 轨迹特征、KG特征、天气特征直接拼接后输入分类器
- **Exp5（弱监督）**: KG特征和天气特征不直接参与分类，而是通过一致性损失约束轨迹表示的学习

#### 调用示例

```python
from predict import TransportationPredictorExp5
import numpy as np

predictor = TransportationPredictorExp5("checkpoints/exp5_model.pth")

trajectory = np.array([...])    # shape: (50, 9)
kg_features = np.array([...])   # shape: (15,)
weather = np.array([...])       # shape: (12,)

label, confidence = predictor.predict(trajectory, kg_features, weather)
```

#### 性能说明

⚠️ **注意**: Exp5的准确率（64.01%）低于Exp4（74.80%），弱监督约束在本任务中效果不佳。该模型主要用于研究验证，不建议用于生产部署。

### 5.3 特征计算工具函数（后端需要实现）

```python
import numpy as np
from math import radians, sin, cos, sqrt, atan2

def haversine_distance(lat1, lon1, lat2, lon2):
    """计算两点间的距离（米）"""
    R = 6371000  # 地球半径（米）
    lat1, lon1, lat2, lon2 = map(radians, [lat1, lon1, lat2, lon2])
    dlat = lat2 - lat1
    dlon = lon2 - lon1
    a = sin(dlat/2)**2 + cos(lat1) * cos(lat2) * sin(dlon/2)**2
    return 2 * R * atan2(sqrt(a), sqrt(1-a))

def calculate_bearing(lat1, lon1, lat2, lon2):
    """计算航向角（度）"""
    lat1, lon1, lat2, lon2 = map(radians, [lat1, lon1, lat2, lon2])
    dlon = lon2 - lon1
    x = sin(dlon) * cos(lat2)
    y = cos(lat1) * sin(lat2) - sin(lat1) * cos(lat2) * cos(dlon)
    bearing = atan2(x, y)
    return (np.degrees(bearing) + 360) % 360

def extract_trajectory_features(gps_points):
    """
    从GPS点列表提取9维轨迹特征

    输入: gps_points = [
        {"latitude": 39.9, "longitude": 116.4, "timestamp": datetime},
        ...
    ]

    输出: numpy array, shape (n_points, 9)
    """
    n = len(gps_points)
    features = np.zeros((n, 9))

    total_distance = 0
    total_time = 0
    prev_speed = 0
    prev_bearing = 0

    for i in range(n):
        point = gps_points[i]
        features[i, 0] = point['latitude']
        features[i, 1] = point['longitude']

        if i == 0:
            features[i, 2:] = 0  # 第一个点没有速度等特征
        else:
            prev = gps_points[i-1]

            # 距离和时间差
            dist = haversine_distance(
                prev['latitude'], prev['longitude'],
                point['latitude'], point['longitude']
            )
            time_diff = (point['timestamp'] - prev['timestamp']).total_seconds()
            time_diff = max(time_diff, 0.1)  # 避免除零

            # 速度和加速度
            speed = dist / time_diff
            acceleration = (speed - prev_speed) / time_diff

            # 方向变化
            bearing = calculate_bearing(
                prev['latitude'], prev['longitude'],
                point['latitude'], point['longitude']
            )
            bearing_change = abs(bearing - prev_bearing)
            if bearing_change > 180:
                bearing_change = 360 - bearing_change

            # 累计值
            total_distance += dist
            total_time += time_diff

            # 填充特征
            features[i, 2] = speed
            features[i, 3] = acceleration
            features[i, 4] = bearing_change
            features[i, 5] = dist
            features[i, 6] = time_diff
            features[i, 7] = total_distance
            features[i, 8] = total_time

            prev_speed = speed
            prev_bearing = bearing

    return features
```

## 七、常见问题FAQ

### Q1: 模型文件从哪里获取？

A: 模型文件 (.pth) 由训练脚本生成，位于各实验的 `checkpoints/` 目录下。我会提供给你们。

### Q2: 轨迹点数量有要求吗？

A: 建议50-100个点。太少（<10个）特征不足，太多（>500个）计算耗时。模型内部会自动进行序列长度规范化。

### Q3: KG特征和天气特征怎么获取？

A: 

- **简化方案**：可以使用默认值（全零向量），主要展示轨迹特征的效果
- **完整方案**：需要调用OSM API获取地理信息，调用天气API获取历史天气

### Q4: 为什么推荐Exp2而不是Exp4？

A: Exp2准确率最高（79.49%），且特征获取相对简单。Exp4增加了天气特征反而导致性能下降。

### Q5: Exp5为什么准确率最低？

A: Exp5采用弱监督约束策略，KG特征和天气特征不直接参与分类，而是通过一致性损失约束轨迹表示的学习。在本任务中，这种策略效果不佳，准确率仅为64.01%。该模型主要用于研究验证，不建议用于生产部署。

### Q6: 预测速度如何？

A: 单条轨迹预测约50-200ms（取决于硬件），可以满足实时交互需求。

### Q7: 需要GPU吗？

A: 不需要，CPU即可运行预测。GPU主要用于训练。
