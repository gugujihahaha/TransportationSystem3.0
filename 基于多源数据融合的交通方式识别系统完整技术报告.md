# 基于多源数据融合的交通方式识别系统 - 完整技术报告

## 📋 报告概述

本报告详细介绍了基于多源数据融合的交通方式识别系统的完整技术实现。系统采用递进式实验设计，通过逐步引入轨迹特征、知识图谱特征和天气特征，系统性地分析了不同特征组合对识别性能的影响。

**报告日期**: 2026年1月14日
**技术栈**: Python, PyTorch, Scikit-learn, NetworkX
**数据集**: GeoLife Trajectories 1.3

## 🔄 数据流

### 1. 数据加载流程

```
GeoLife原始数据
    ↓
BaseGeoLifePreprocessor (基础预处理)
    ↓
两阶段数据清洗
    ├─ 第一阶段：基础预处理
    │  • 填充缺失特征
    │  • 统一轨迹序列长度（默认50）
    │  • 快速平滑速度/加速度异常点
    └─ 第二阶段：深度清洗
       • 剔除异常轨迹段（速度/加速度/方向过大）
       • 插值缺失点，处理时间跳变
       • 平滑异常方向（bearing）
       • 统一序列长度
       • 记录清洗统计信息
    ↓
9维轨迹特征提取
    ↓
实验适配器 (Exp1/2/3/4 Adapter)
    ↓
序列长度规范化
    ↓
知识图谱特征提取 (Exp2/3/4)
    ↓
天气特征提取 (Exp4)
    ↓
最终特征向量
    ↓
模型训练/评估
```

#### 2.4 清洗模式

系统支持三种清洗模式：

- **balanced**: 平衡模式，默认模式，在数据质量和样本保留之间取得平衡
- **strict**: 严格模式，更激进的清洗策略，丢弃更多异常样本
- **lenient**: 宽松模式，保留更多样本，但可能包含一些异常数据

划分方法：分层采样 (stratified sampling)

目的：保证各类别在训练/验证/测试集中的比例一致

#### 1.2 轨迹清洗模块 (TrajectoryCleaner)

**功能**: 对轨迹数据进行两阶段清洗，确保数据质量

**核心特性**:
- 交通方式自适应阈值：不同交通工具使用不同的物理约束
- 分层清洗策略：物理异常检测 → 统计异常检测 → 平滑优化
- 保留率控制：确保数据质量的同时最大化样本保留
- 智能插值：基于运动模式的缺失值填充

**清洗参数**:
```python
# 交通方式特定的速度上限 (m/s)
speed_limits = {
    'Walk': 2.5,        # 步行: 2.5 m/s (~9 km/h)
    'Bike': 8.0,        # 自行车: 8 m/s (~28.8 km/h)
    'Bus': 22.0,        # 公交: 22 m/s (~79.2 km/h)
    'Car & taxi': 35.0,  # 汽车: 35 m/s (~126 km/h)
    'Train': 45.0,       # 火车: 45 m/s (~162 km/h)
    'Subway': 30.0,     # 地铁: 30 m/s (~108 km/h)
    'Airplane': 150.0,   # 飞机: 150 m/s (~540 km/h)
}

# 交通方式特定的加速度上限 (m/s²)
accel_limits = {
    'Walk': 2.0,        # 步行: 2 m/s²
    'Bike': 3.0,        # 自行车: 3 m/s²
    'Bus': 4.0,        # 公交: 4 m/s²
    'Car & taxi': 5.0,  # 汽车: 5 m/s²
    'Train': 2.0,       # 火车: 2 m/s² (加速慢)
    'Subway': 3.0,     # 地铁: 3 m/s²
    'Airplane': 8.0,    # 飞机: 8 m/s²
}

# 通用参数
max_time_gap = 300.0        # 最大允许时间间隔 (秒)
max_bearing_change = 150.0   # 最大允许方向变化 (度)
min_segment_length = 10      # 最小轨迹段长度
max_outlier_ratio = 0.25     # 最大异常点比例
```

**清洗阶段**:

**阶段1: 物理异常检测与修复**
- 检测速度、加速度、方向变化异常
- 基于交通工具物理特性进行判断
- 使用局部中值滤波修复异常点

**阶段2: 时间连续性处理**
- 检测大时间间隔 (> max_time_gap)
- 小间隔 (<60s): 线性插值
- 大间隔 (>60s): 标记但不插值

**阶段3: 轨迹平滑与优化**
- 对速度、加速度使用Savitzky-Golay滤波器平滑
- 对方向变化使用移动平均平滑
- 保留轨迹的整体趋势

**清洗统计信息**:

### 2. 训练循环

```python
for epoch in range(epochs):
    # 训练阶段
    model.train()
    for batch in train_loader:
        # 前向传播
        logits = model(features)
        loss = criterion(logits, labels)

        # 反向传播
        optimizer.zero_grad()
        loss.backward()
        optimizer.step()

    # 验证阶段
    model.eval()
    val_loss, val_acc = evaluate(model, val_loader)

    # Early Stopping
    if val_loss < best_val_loss:
        save_model()
        best_val_loss = val_loss
        epochs_no_improve = 0
    else:
        epochs_no_improve += 1
        if epochs_no_improve >= patience:
            break
```

## 📈 实验结果对比

### 整体性能

| 实验 | 特征组合 | 融合策略 | 准确率 | 加权F1 | 宏平均F1 |
|------|---------|----------|--------|---------|----------|
| Exp1 | 轨迹 (9维) | 无融合 | 69.60% | 0.6818 | 0.5291 |
| Exp2 | 轨迹+基础KG (20维) | 硬拼接 | 79.49% | 0.7916 | 0.6540 |
| Exp3 | 轨迹+增强KG (24维) | 硬拼接 | 78.25% | 0.7793 | 0.6287 |
| Exp4 | 轨迹+KG+天气 (36维) | 硬拼接 | 74.80% | 0.7373 | 0.5073 |
| Exp5 | 轨迹+KG+天气 (36维) | 弱监督约束 | 64.01% | 0.6131 | 0.3838 |

### 性能提升分析

**Exp1 → Exp2**: +9.89% 准确率
- 知识图谱特征贡献显著
- 所有类别F1分数大幅提升
- Subway识别改善最明显 (F1: 0.5134 → 0.8502)
- 加权F1从0.6818提升至0.7916

**Exp2 → Exp3**: -1.24% 准确率
- 增强特征导致边际递减
- 可能存在特征冗余
- Walk召回率略有提升
- 加权F1从0.7916下降至0.7793

**Exp3 → Exp4**: -3.45% 准确率
- 天气特征贡献有限
- 可能引入噪声
- Train识别性能显著下降 (F1: 0.5789 → 0.0000)
- 加权F1从0.7793下降至0.7373

**Exp4 → Exp5**: -10.79% 准确率
- 弱监督约束效果不佳
- Subway识别严重受损 (F1: 0.6614 → 0.2667)
- 轨迹表示与上下文表示对齐困难
- 加权F1从0.7373下降至0.6131

- 📉 **Subway识别严重受损**: F1从0.6614降至0.2667，召回率从0.7411降至0.1786
- 🔍 **融合策略对比**: 弱监督约束在本任务中不如硬拼接有效

- 实验结果公平可比
